## 一、硬件配置

| 组件 | 型号/规格 | 接口 |
|------|----------|------|
| MCU | STM32F103C8T6 | — |
| 电机驱动 | TB6612 或同类 H 桥 | PA3/PA4 左方向, PA0/PA5 右方向 |
| PWM | TIM2 CH2(右轮 PA1), CH3(左轮 PA2) | 频率 = 72MHz / (36×100) = 20kHz |
| 红外循迹 | 8 路红外对管（模拟量整形为数字量） | 见下方引脚表 |
| 陀螺仪 | MPU6050 (I2C + DMP) | 用于角度 PD 走直线，非循迹必须 |

### 八路红外传感器引脚

```
传感器编号    物理位置      GPIO      在状态字节中的位
─────────────────────────────────────────────────
红外1 (最左)   左4         PA15       bit7 (最高位)
红外2          左3         PA8        bit6
红外3          左2         PA11       bit5
红外4          左1         PB15       bit4
红外5          右1         PA7        bit3
红外6          右2         PB14       bit2
红外7          右3         PA6        bit1
红外8 (最右)   右4         PB13       bit0 (最低位)
```

- 所有引脚配置为 **上拉输入** (`GPIO_Mode_IPU`)
- 黑线 = 低电平 → 读到 0，但代码中反逻辑处理（见下文）
- **注意**：PA15 默认是 JTDI，需禁掉 JTAG：`GPIO_PinRemapConfig(GPIO_Remap_SWJ_JTAGDisable, ENABLE)`

## 二、数据流全景（核心！）

```
main() 循环
  │
  ├─ Get_Infrared_State()          // 8个传感器 → 8bit状态字节
  │     │
  │     └─ Track_err()             // 状态字节 → 误差值（-6 ~ +6）
  │           │
  │           └─ PID_out(err, 0)   // PD控制器 → 修正量（-25 ~ +25）
  │                 │
  │                 └─ Final_Speed(pid_out, base_speed)  // 修正量 → 左右轮速度
  │                       │
  │                       └─ Set_Speed(left, right)      // 速度值 → GPIO + PWM
  │                             │
  │                             ├─ 左轮: PA3/PA4 方向 + TIM2_CH3 PWM
  │                             └─ 右轮: PA0/PA5 方向 + TIM2_CH2 PWM
  │
  └─ pid_angle2(target, yaw)      // 【备用】陀螺仪角度PD，用于无线的直线段
```

**两条控制路径**：
- **有线可循时** → 走 `Track_err → PID_out → Final_Speed` 路径
- **无线（全白）时** → 走 `pid_angle2` 陀螺仪保方向

## 三、逐层代码拆解

### 第 1 层：读取 8 路红外 → 状态字节

```c
// TRACK.h 中的引脚宏（按你自己的硬件改！）
#define TRACK_INFRARED_PIN_1   GPIO_Pin_15   // 红外1, 左4
#define TRACK_INFRARED_PIN_2   GPIO_Pin_8    // 红外2, 左3
#define TRACK_INFRARED_PIN_3   GPIO_Pin_11   // 红外3, 左2
#define TRACK_INFRARED_PIN_4   GPIO_Pin_15   // 红外4, 左1 (在PB口!)
#define TRACK_INFRARED_PIN_5   GPIO_Pin_7    // 红外5, 右1
#define TRACK_INFRARED_PIN_6   GPIO_Pin_14   // 红外6, 右2 (在PB口!)
#define TRACK_INFRARED_PIN_7   GPIO_Pin_6    // 红外7, 右3
#define TRACK_INFRARED_PIN_8   GPIO_Pin_13   // 红外8, 右4 (在PB口!)
```

```c
// 返回 8 位状态字节
// 每一位: 1 = 传感器在黑线上, 0 = 传感器在白区
u8 Get_Infrared_State(void)
{
    u8 bit7 = GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_15) << 7;
    u8 bit6 = GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_8)  << 6;
    u8 bit5 = GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_11) << 5;
    u8 bit4 = GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_15) << 4;
    u8 bit3 = GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_7)  << 3;
    u8 bit2 = GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_14) << 2;
    u8 bit1 = GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_6)  << 1;
    u8 bit0 = GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_13) << 0;

    return bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0;
}
```

**为什么用 OR 拼而不是直接移位赋值？** 因为每路都独立读 GPIO，需要最后合并。

### 第 2 层：状态字节 → 误差值（查表法）

这是整个巡线的**核心映射表**。原理：

- **error > 0** → 线在车身左侧 → 小车偏右了 → 需要左转
- **error < 0** → 线在车身右侧 → 小车偏左了 → 需要右转
- **error 绝对值** → 偏离程度（0=正中, 2=微偏, 4=中偏, 6=严重偏）

```c
float Track_err(void)
{
    u8 state = Get_Infrared_State();

    // ★ 防抖：连续2次读到相同状态才更新 error
    static u8 last_state = 0;
    static u8 same_cnt = 0;

    if (state == last_state) {
        same_cnt++;
    } else {
        same_cnt = 0;
        last_state = state;
    }
    if (same_cnt < 2) {
        static float last_error = 0;
        return last_error;  // 不够2次，返回旧值
    }

    float error;
    switch (state)
    {
        // ===== 正中间（理想状态）=====
        case 0x18:  // 0001 1000  中间4灯在线
        case 0x7E:  // 0111 1110  十字路口（全黑）
        case 0x00:  // 0000 0000  完全脱线
            error = 0;  break;

        // ===== 线偏左（小车偏右，需左转）→ 正error =====
        case 0x30:  // 0011 0000  线偏左一点
        case 0x20:  // 0010 0000
            error = 2;  break;
        case 0x40:  // 0100 0000  线偏左较多
        case 0x60:  // 0110 0000
            error = 4;  break;
        case 0x80:  // 1000 0000  线在最左边
        case 0xC0:  // 1100 0000
        case 0xE0:  // 1110 0000
            error = 6;  break;

        // ===== 线偏右（小车偏左，需右转）→ 负error =====
        case 0x0C:  // 0000 1100  线偏右一点
        case 0x04:  // 0000 0100
            error = -2;  break;
        case 0x06:  // 0000 0110  线偏右较多
        case 0x0E:  // 0000 1110
            error = -4;  break;
        case 0x01:  // 0000 0001  线在最右边
        case 0x03:  // 0000 0011
        case 0x07:  // 0000 0111
        case 0x0F:  // 0000 1111
            error = -6;  break;

        default:
            break;  // 未定义状态保持原值
    }

    last_state = state;
    same_cnt = 0;
    return error;
}
```

**查表设计逻辑**（给你理解用，不用背）：

```
传感器物理位置：  [红外1] [2] [3] [4] [5] [6] [7] [红外8]
                    左 ← → 右
bit 位置：         bit7  bit6 bit5 bit4 bit3 bit2 bit1 bit0

"线在左边" 的表现：左边几个传感器=1，右边=0
  例: 1100 0000 → error = +6（严重偏左，小车需大左转）

"线在右边" 的表现：右边几个传感器=1，左边=0
  例: 0000 0011 → error = -6（严重偏右，小车需大右转）
```

### 第 3 层：PD 控制器

**位置式 PD**（无积分项 I），公式：

$$out = K_p \cdot err + K_d \cdot (err - err_{last})$$

```c
#define KP  2.5f   // 比例系数
#define KD  3.5f   // 微分系数

int PID_out(float error, int target)   // target 传 0（目标是 error=0，即线在正中）
{
    static int last_err = 0;
    int err = (int)error;
    int diff = err - last_err;

    // ★ 首次调用防冲击：微分项减半，避免初始瞬间大转向
    if (last_err == 0 && err != 0) {
        diff = err / 2;
    }

    int out = (int)(KP * err + KD * diff);

    // ★ 输出限幅（避免满偏堵转）
    if (out > 25)  out = 25;
    if (out < -25) out = -25;

    last_err = err;
    return out;
}
```

**参数含义**：
- **KP=2.5**：偏离 1 个单位 → 输出 2.5 的修正，偏离越大修正越强
- **KD=3.5**：微分项抑制震荡，误差变化越快阻尼越大
- **限幅 ±25**：防止 PID 输出超出 PWM 量程（PWM 范围 -50~+50，base_speed 一般 25，所以限幅 25 保证单轮最低速度 ≥0）

### 第 4 层：修正量 → 左右轮速度

```c
void Final_Speed(int pid_out, int base_speed)
{
    int left  = base_speed - pid_out;   // pid_out>0时左轮减速 → 左转
    int right = base_speed + pid_out;   // pid_out>0时右轮加速 → 左转

    // 限幅：最低5防堵转，最高40防飞车
    #define MIN_SPEED 5
    #define MAX_SPEED 40

    if (left  < MIN_SPEED) left  = MIN_SPEED;
    if (left  > MAX_SPEED) left  = MAX_SPEED;
    if (right < MIN_SPEED) right = MIN_SPEED;
    if (right > MAX_SPEED) right = MAX_SPEED;

    Set_Speed(left, right);
}
```

**差速转向原理**：

| pid_out | 含义 | 左轮 | 右轮 | 效果 |
|---------|------|------|------|------|
| 0 | 正中 | 25 | 25 | 直走 |
| +6 | 需左转 | 19 | 31 | **左转** |
| −6 | 需右转 | 31 | 19 | **右转** |

### 第 5 层：速度 → 电机执行

```c
// Motor.c
void Set_Speed(int motor_l, int motor_r)
{
    // 左电机
    if (motor_l >= 0) {
        GPIO_SetBits  (GPIOA, GPIO_Pin_3);   // PA3 正转方向
        GPIO_ResetBits(GPIOA, GPIO_Pin_4);   // PA4 反转方向
        TIM_SetCompare3(TIM2, motor_l);      // CH3 PWM 占空比
    } else {
        GPIO_SetBits  (GPIOA, GPIO_Pin_4);
        GPIO_ResetBits(GPIOA, GPIO_Pin_3);
        TIM_SetCompare3(TIM2, -motor_l);     // 取绝对值
    }

    // 右电机
    if (motor_r >= 0) {
        GPIO_SetBits  (GPIOA, GPIO_Pin_0);   // PA0 正转方向
        GPIO_ResetBits(GPIOA, GPIO_Pin_5);   // PA5 反转方向
        TIM_SetCompare2(TIM2, motor_r);      // CH2 PWM 占空比
    } else {
        GPIO_SetBits  (GPIOA, GPIO_Pin_5);
        GPIO_ResetBits(GPIOA, GPIO_Pin_0);
        TIM_SetCompare2(TIM2, -motor_r);
    }
}
```

速度范围：**-50 ~ +50**（PWM 周期 100，占空比 0~99，正数前进负数后退）

## 四、PWM 初始化（供参考）

```c
// TIM2: CH2=PA1(右轮PWM), CH3=PA2(左轮PWM)
// 72MHz / (36 * 100) = 20kHz, 分辨率 100 级
void PWM_Init(void)
{
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE);
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);

    // PA1, PA2 → 复用推挽输出
    GPIO_InitTypeDef g = {
        .GPIO_Mode = GPIO_Mode_AF_PP,
        .GPIO_Pin  = GPIO_Pin_1 | GPIO_Pin_2,
        .GPIO_Speed = GPIO_Speed_50MHz
    };
    GPIO_Init(GPIOA, &g);

    TIM_TimeBaseInitTypeDef t = {
        .TIM_Period = 100 - 1,       // ARR
        .TIM_Prescaler = 36 - 1,     // PSC
        .TIM_ClockDivision = TIM_CKD_DIV1,
        .TIM_CounterMode = TIM_CounterMode_Up
    };
    TIM_TimeBaseInit(TIM2, &t);

    TIM_OCInitTypeDef oc = {
        .TIM_OCMode = TIM_OCMode_PWM1,
        .TIM_OCPolarity = TIM_OCPolarity_High,
        .TIM_OutputState = TIM_OutputState_Enable,
        .TIM_Pulse = 0               // 初始占空比 0
    };
    TIM_OC3Init(TIM2, &oc);  // 左轮 CH3
    TIM_OC2Init(TIM2, &oc);  // 右轮 CH2
    TIM_Cmd(TIM2, ENABLE);
}
```

## 五、主循环调用模板

```c
// 在你的 main() 或 RTOS 线程中，每 10ms 调用一次：

// 1. 读陀螺仪（可选，用于直线段角度保持）
float pitch, roll, yaw;
while (MPU6050_DMP_Get_Data(&pitch, &roll, &yaw) != 0);  // 重试直到成功

// 2. 判断当前是否有线可循
u8 ir_state = Get_Infrared_State();

if (ir_state != 0) {
    // ===== 有线：循迹 PD 模式 =====
    float err = Track_err();
    int   pid = PID_out(err, 0);
    Final_Speed(pid, 25);   // base_speed=25
} else {
    // ===== 无线：陀螺仪角度保持模式 =====
    int yaw_int = (int)((yaw < 0 ? yaw + 360 : yaw) + 0.5f);
    int pid = pid_angle2(0, yaw_int);   // 目标角度 0°
    int left  = 30 - pid;    // 角度模式下 base_speed 可稍大
    int right = 30 + pid;
    Limit(&left, &right);    // 限幅 ±50
    Set_Speed(left, right);
}
```

## 六、调参指南（告诉另一个 AI 怎么调）

| 参数 | 默认值 | 作用 | 调大后果 | 调小后果 |
|------|--------|------|---------|---------|
| `KP` | 2.5 | 比例响应强度 | 修正过猛，S 形摇摆 | 修正无力，线跑了拉不回来 |
| `KD` | 3.5 | 阻尼/微分 | 响应迟缓，转弯"肉" | 剧烈震荡，左右摆 |
| `base_speed` | 25 | 基础前进速度 | 过弯冲出去 | 太慢，效率低 |
| `MIN_SPEED` | 5 | 最低轮速 | 低速仍堵转（无意义） | 弯内侧轮停转，转弯生硬 |
| `MAX_SPEED` | 40 | 最高轮速 | 差速不够大 | 差速够但极速受限 |
| 防抖 `same_cnt` | ≥2 | 滤波强度 | 响应迟钝 | 毛刺多，抖动 |
| PID 输出限幅 | ±25 | 最大修正量 | 转弯猛但可能甩尾 | 大弯转不过来 |

**调参顺序建议**：先调 KP（原地晃车身看响应）→ 再调 KD（走直线看震荡）→ 最后调 base_speed（跑全程看速度）。

## 七、常见坑（踩过的）

1. **PA15 不工作** → 禁掉 JTAG：`GPIO_PinRemapConfig(GPIO_Remap_SWJ_JTAGDisable, ENABLE)`
2. **传感器读数反了** → 检查 `GPIO_Mode_IPU` vs `IPD`，有些模块黑线=低、白=高
3. **PID 输出符号反了** → 表现是越巡越偏，把 `Final_Speed` 里的 `±` 反过来
4. **静态变量不重置** → 脱线后重新上线前，调用 `PID_out(0,0)` 重置 `last_err`
5. **DMP 读取失败** → `MPU6050_DMP_Get_Data` 返回非 0 时需循环重试，否则读到旧数据
6. **ARMCC C89 限制** → Keil 的 ARMCC 编译器要求变量声明在块开头，不能随处声明

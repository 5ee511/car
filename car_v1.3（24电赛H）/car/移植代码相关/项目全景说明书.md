## 三、主程序框架 —— 三层嵌套状态机

```c
main()
  │
  ├─ 初始化所有外设（Track/LED/Buzzer/Motor/Key/OLED/MPU6050）
  │
  └─ while(1) 主循环
       │
       ├─ 读取 DMP（获取 Pitch/Roll/Yaw）
       ├─ OLED 显示调试信息（Yaw, all_state, step, 红外状态）
       │
       ├─ 【第一层】按键读取 → 修改 all_state
       │    ├─ 按键1：切换 low 4-bit → 0x01/0x02/0x03/0x04 循环
       │    └─ 按键2：设置 bit4 → 0x11/0x12/0x13/0x14 启动
       │
       └─ 【第二层】switch(all_state)
            │
            ├─ case 17 (0x11) → 第一问
            ├─ case 18 (0x12) → 第二问
            ├─ case 19 (0x13) → 第三问
            ├─ case 20 (0x14) → 第四问
            │
            └─ 每问内部有【第三层】switch(step) 子状态机
```

**关键设计**：
- `all_state` 的低 4 位（`& 0x000F`）= 赛题编号（1~4）
- `all_state` 的 bit4（`& 0x0010`）= 启动标志，按键2置位
- 启动后 `all_state` = `0x10 + 编号`，即 17/18/19/20
- `step` = 每道赛题内的子步骤（1, 2, 3, ...）

## 四、两套 PD 控制系统（项目的核心算法）

本项目的运动控制只有两种模式，交替使用：

### 模式 A：红外循迹 PD（有线可循时使用）

**调用链**：
```
Get_Infrared_State()   → 8位传感器状态
        ↓
Track_err()            → 查表映射为误差值 (-6 ~ +6)
        ↓
PID_out(error, 0)      → PD 控制器输出修正量 (-25 ~ +25)
        ↓
Final_Speed(pid, base) → 修正量转为左右轮速度 (5 ~ 40)
        ↓
Set_Speed(L, R)        → GPIO方向 + TIM2 PWM 占空比
```

- **Kp=2.5, Kd=3.5**，位置式 PD，无积分项
- **base_speed=25**（大部分场景），base_speed=22（部分过弯场景减速）
- PID 输出限幅 ±25，轮速限幅 [5, 40]

### 模式 B：陀螺仪角度 PD（无线全白时保方向）

**调用链**：
```
MPU6050_DMP_Get_Data()  → Yaw (-180° ~ +180°)
        ↓
Yaw → 0~360° 转换
        ↓
pid_angle2(target, yaw) → 角度 PD 输出修正量 (-20 ~ +20)
        ↓
aleft = base - pid      → 直接算左右轮速
aright = base + pid
        ↓
Limit(&aleft, &aright)  → 限幅 ±50
        ↓
Set_Speed(aleft, aright)
```

- **ka2p=0.5, ka2d=3.5**，角度 PD
- 角度误差计算使用"最短路径"逻辑（`err = target - yaw`，>180° 则 -360°，<-180° 则 +360°）
- 角度 PID 输出限幅 ±20
- **base_speed=30**（角度模式），比循迹模式稍快

## 五、四道赛题完整流程

### 第一问 (all_state=17)：沿直线循迹走到终点

**单阶段，无子步骤**。

```
出发（红外全白，无线）
  │
  └─ while(红外全白)：pid_angle2(0°, yaw)  ← 陀螺仪保持 0° 方向直走
       │
       直到红外踩到终点线 → 停车 → LED蜂鸣器翻转 → 结束
```

- 目标角度：**0°**（正前方）
- base_speed：**30**

---

### 第二问 (all_state=18)：循线+过路口

**5 个子步骤**。

```
step 1：直线段1（角度模式，目标0°）
  红外全白 → pid_angle2(0°, yaw) 直走 → 直到红外踩到线
  ↓ 停车 → 重置PID → step 2

step 2：循线段1（循迹模式）
  红外有线 → Track_err/PID_out/Final_Speed(pid, 25)
  ↓ 直到红外全白（脱线）→ 停车 → step 3

step 3：直线段2（角度模式，目标168°）
  红外全白 → pid_angle2(168°, yaw) 直走 → 直到红外踩到线
  ↓ 停车 → 重置PID → step 4

step 4：循线段2（循迹模式）
  红外有线 → Track_err/PID_out/Final_Speed(pid, 25)
  ↓ 直到红外全白（脱线）→ step 5

step 5：停车，结束
```

**规律总结**：角度模式走空地 → 遇线切循迹 → 脱线切角度 → 再遇线切循迹 → 到达终点。

---

### 第三问 (all_state=19)：循线+过路口（不同角度）

**5 个子步骤，与第二问结构完全相同，但角度参数不同**。

```
step 1：角度模式 → pid_angle2(50°, yaw)    ← 目标角 50°
step 2：循迹模式 → Final_Speed(pid, 25)
step 3：角度模式 → pid_angle2(130°, yaw)   ← 目标角 130°
step 4：循迹模式 → Final_Speed(pid, 22)    ← 注意减速到22
step 5：停车，结束
```

---

### 第四问 (all_state=20)：多圈循环（三圈）

**17 个子步骤，最复杂**。

#### 第一圈 (step 1~4)

```
step 1：角度模式 → pid_angle2(50°, yaw)    ← 斜线1，目标角50°
step 2：循迹模式 → Final_Speed(pid, 25)    ← 循线段
step 3：角度模式 → pid_angle2(129°, yaw)   ← 斜线2，目标角129°  (注意不是130°)
step 4：循迹模式 → Final_Speed(pid, 22)    ← 循线段（减速22）
```

#### 第二圈 (step 5~8)

```
step 5：角度模式 → pid_angle2(59°, yaw)    ← 斜线1，微调到59°
step 6：循迹模式 → Final_Speed(pid, 25)
step 7：角度模式 → pid_angle2(137°, yaw)   ← 斜线2，微调到137°
step 8：循迹模式 → Final_Speed(pid, 22)
```

#### 第三圈 (step 9~12)

```
step 9： 角度模式 → pid_angle2(65°, yaw)    ← 斜线1，微调到65°
step 10：循迹模式 → Final_Speed(pid, 25)
step 11：角度模式 → pid_angle2(145°, yaw)   ← 斜线2，微调到145°
step 12：循迹模式 → Final_Speed(pid, 22)
```

#### 第四圈 (step 13~16)

```
step 13：角度模式 → pid_angle2(71°, yaw)    ← 斜线1，微调到71°
step 14：循迹模式 → Final_Speed(pid, 25)
step 15：角度模式 → pid_angle2(150°, yaw)   ← 斜线2，微调到150°
step 16：循迹模式 → Final_Speed(pid, 22)
```

step 17：停车，结束。

**角度递增规律**：

| 圈数 | 斜线1 目标角 | 斜线2 目标角 |
|------|-------------|-------------|
| 第1圈 | 50° | 129° |
| 第2圈 | 59° (+9) | 137° (+8) |
| 第3圈 | 65° (+6) | 145° (+8) |
| 第4圈 | 71° (+6) | 150° (+5) |

角度逐圈递增，因为小车过弯后会有角度累积偏移，需要微调补偿。

## 六、每个 Step 的标准代码模板

AI 助手注意：项目中每个 step 的结构高度统一，以下是模板：

```
【角度模式 step 模板】（走空地/斜线）
  1. delay_ms(100或300)           // 短暂延时，让电机状态稳定
  2. while (Get_Infrared_State()==0)   // 红外全白 = 还在空地上
     {
       读 DMP 获取 Yaw
       Yaw 转为 0~360°
       pid_out = pid_angle2(目标角度, yaw_int)
       aleft  = 25或30 - pid_out
       aright = 25或30 + pid_out
       Limit(&aleft, &aright)
       Set_Speed(aleft, aright)
     }
  3. // 退出循环 = 红外踩到线了
  4. if (Get_Infrared_State()!=0)
     {
       Set_Speed(0, 0)            // 停车
       delay_ms(50或100)           // 等车停稳
       PID_out(0, 0)              // ★ 重置循迹PID的last_err
       find_err = 0.0f            // ★ 重置误差
       step = 下一个编号            // 推进
     }

【循迹模式 step 模板】（跟着线走）
  1. LED1_Turn() + buzzer_turn()   // 进入新阶段，声光提示
  2. while (Get_Infrared_State()!=0)   // 红外有线 = 还在线上
     {
       读 DMP 获取 Yaw
       find_err = Track_err()
       pid_out  = PID_out(find_err, 0)
       Final_Speed(pid_out, 25或22)   // 循迹速度一般25，过弯减速用22
     }
  3. // 退出循环 = 红外全白 = 脱线了
  4. if (Get_Infrared_State()==0)
     {
       Set_Speed(0, 0)
       delay_ms(50)
       PID_out(0, 0)
       find_err = 0.0f
       step = 下一个编号
     }

【结束 step 模板】
  while(1)
  {
    Set_Speed(0, 0)
    step = 999
  }
```

## 八、常见注意事项与坑

1. **PA15 不工作** → `GPIO_PinRemapConfig(GPIO_Remap_SWJ_JTAGDisable, ENABLE)` 禁掉 JTAG，否则 PA15/JTDI 无法当 GPIO
2. **DMP 读取失败** → `MPU6050_DMP_Get_Data()` 返回非零时必须在 while 中循环重试，I2C 时序敏感，偶尔丢包
3. **DMP 初始化失败** → `main()` 开头的 `MPU6050_DMP_Init()` 可能失败，实测会持续重试直到成功
4. **ARMCC C89 限制** → 变量声明必须在代码块开头，不能 `int x = ...` 写在 `x = func()` 之前之外的任意位置
5. **PID 状态未重置** → 从角度模式切换到循迹模式前，必须调 `PID_out(0,0)` 重置 `last_err`，否则首次循迹的微分项会炸
6. **防抖不充分** → `Track_err()` 中 `same_state_cnt < 2` 意味着要求连续 2 次读到相同状态才更新，可调大此阈值
7. **OLED 和 MPU6050 共用 I2C** → 都是软件 I2C(PB10/PB11)，分时复用，OLED 刷新和 DMP 读取交替进行，互不影响
8. **中文注释乱码** → 源码编码为 GB2312/GBK，在 UTF-8 环境打开会显示乱码，但不影响编译

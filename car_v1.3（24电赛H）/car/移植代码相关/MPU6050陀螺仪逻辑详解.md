## 一、文件架构 —— 四层剥离

本项目 MPU6050 驱动由 **4 个文件** 组成，严格分层：

```
Hardware/Mpu6050/
│
├── MPU6050_I2C.c/h          ← 第1层：软件 I2C 物理层
│     （引脚 PB6=SCL, PB7=SDA, 4μs延时, 开漏输出模拟）
│
├── mpu6050.c/h               ← 第2层：MPU6050 寄存器读写 + 基础配置
│     （I2C 读写函数、初始化、量程设置、原始数据读取）
│
├── inv_mpu.c/h               ← 第3层：InvenSense 官方 DMP 移植层
│     （mpu_init、自检、传感器配置、DMP 状态机）
│
└── inv_mpu_dmp_motion_driver.c/h  ← 第4层：DMP 固件 + 姿态解算
      （DMP 固件加载、dmp_read_fifo、四元数→欧拉角）
```

**调用关系（从上到下）**：

```
main.c
  ├─ MPU6050_Init()          ← 第2层：基础初始化（唤醒、量程、采样率）
  ├─ MPU6050_DMP_Init()      ← 第3层：DMP 初始化（加载固件、配置FIFO、自检）
  └─ MPU6050_DMP_Get_Data()  ← 第3层：读取姿态（读FIFO → 四元数 → 欧拉角）
       └─ dmp_read_fifo()    ← 第4层：从 MPU6050 内部 DMP FIFO 读取原始四元数
            └─ i2c_read()    ← 最终落到第1层 I2C
```

## 二、第1层：软件 I2C —— 最底层通信

### 引脚配置

| 信号 | GPIO | 宏别名 | 说明 |
|------|------|--------|------|
| SCL | PB6 | `MPU6050_IIC_SCL` | 时钟线，开漏输出 |
| SDA | PB7 | `MPU6050_IIC_SDA` | 数据线，输入输出动态切换 |
| 延时 | — | `MPU6050_IIC_delay_4us()` | 每次电平翻转延时 4μs，约 100kHz |

### 为什么用软件 I2C 而不用 STM32 硬件 I2C？

- STM32F1 的硬件 I2C 有著名的"锁死"bug（BUSY 位卡死），在电赛这种需要高可靠性的场景下，软件模拟更稳定
- PB6/PB7 和 OLED 的 PB10/PB11 不冲突（不同的引脚），所以可以各自独立

### 核心模式切换：SDA 输入/输出动态切换

```c
// 发数据时：SDA 切为输出
void MPU6050_IIC_SDA_IO_OUT(void) {
    My_GPIO_Init(GPIOB, PB7, GPIO_TW_OUT, GPIO_P_NO, GPIO_50MHz);  // 推挽输出
}

// 收数据时：SDA 切为输入
void MPU6050_IIC_SDA_IO_IN(void) {
    My_GPIO_Init(GPIOB, PB7, GPIO_FK_IN, GPIO_P_UP, GPIO_50MHz);   // 上拉输入
}
```

> `My_GPIO_Init` 是本项目封装的简化 GPIO 初始化函数（定义在 `sys.c`），本质是调用 STM32 的 `GPIO_Init()`，但用枚举值代替了结构体。

### I2C 基本时序函数

```c
void MPU6050_IIC_Start(void);     // 起始信号: SCL高→SDA拉低
void MPU6050_IIC_Stop(void);      // 停止信号: SCL高→SDA拉高
void MPU6050_IIC_Send_Byte(u8);   // 发送1字节(MSB先)，第9个时钟读ACK
u8   MPU6050_IIC_Read_Byte(u8 ack);// 接收1字节，ack=1发NACK(最后一字节),ack=0发ACK
```

**标准 I2C 写时序**：`Start → 设备地址(W) → 寄存器地址 → 数据1...数据N → Stop`
**标准 I2C 读时序**：`Start → 设备地址(W) → 寄存器地址 → Restart → 设备地址(R) → 数据1...数据N → Stop`

### 容错：ACK 超时检测

```c
u8 MPU6050_IIC_Read_Ack(void) {
    u8 ucErrTime = 0;
    // ... SCL拉高后等待SDA被从机拉低
    while(MPU6050_IIC_SDA_IN) {   // SDA一直高 = 从机没应答
        ucErrTime++;
        if (ucErrTime > 250) {     // 超时250次 ≈ 1ms
            MPU6050_IIC_Stop();    // 发停止信号释放总线
            return 1;              // 返回失败
        }
    }
    return 0;
}
```

## 三、第2层：MPU6050 寄存器操作

### I2C 地址

MPU6050 的 7 位地址固定为 **0x68**（AD0 引脚接地）。发送时左移 1 位：

```c
#define MPU_ADDR  0x68
// 实际发送: 写=0xD0, 读=0xD1
addr = addr << 1;   // 在 mpu6050_write/read 内部处理
```

### 单寄存器读写（封装）

```c
void mpu6050_write_reg(u8 reg, u8 dat) {
    mpu6050_write(MPU_ADDR, reg, 1, &dat);   // 写1字节到指定寄存器
}
u8 mpu6050_read_reg(u8 reg) {
    u8 dat;
    mpu6050_read(MPU_ADDR, reg, 1, &dat);    // 从指定寄存器读1字节
    return dat;
}
```

### 多字节连续读写

```c
// 批量写：常用于配置连续寄存器（如 FIFO、DMP 固件加载）
u8 mpu6050_write(u8 addr, u8 reg, u8 len, u8* buf);
// 批量读：常用于读取连续数据寄存器（如加速度3轴=6字节）
u8 mpu6050_read (u8 addr, u8 reg, u8 len, u8* buf);
```

### 基础初始化 `MPU6050_Init()`

```c
void MPU6050_Init(void) {
    MPU6050_IIC_IO_Init();                     // 1. 初始化 I2C 引脚

    mpu6050_write_reg(PWR_MGMT_1, 0x80);       // 2. 复位 MPU6050
    delay_ms(100);                              //    等复位完成

    mpu6050_write_reg(PWR_MGMT_1, 0x00);       // 3. 唤醒（退出睡眠模式）

    MPU_Set_Gyro_Fsr(3);                       // 4. 陀螺仪量程：±2000dps
    MPU_Set_Accel_Fsr(0);                      // 5. 加速度量程：±2g
    MPU_Set_Rate(200);                         // 6. 采样率：200Hz（自检用）
    //  内部自动设置 DLPF = 采样率/2 = 100Hz

    mpu6050_write_reg(MPU_INT_EN_REG, 0x00);   // 7. 关闭所有中断
    mpu6050_write_reg(MPU_USER_CTRL_REG, 0x00);// 8. 关闭 I2C 主模式
    mpu6050_write_reg(MPU_FIFO_EN_REG, 0x00);  // 9. 关闭 FIFO
    mpu6050_write_reg(MPU_INTBP_CFG_REG, 0x80);// 10. INT 引脚低电平有效

    // 11. 验证设备 ID（确认通信正常）
    u8 res = mpu6050_read_reg(MPU_DEVICE_ID_REG);  // 0x75
    if (res == MPU_ADDR) {   // 读到 0x68 = 正确
        mpu6050_write_reg(PWR_MGMT_1, 0x01);   // 时钟源选 PLL X 轴
        mpu6050_write_reg(PWR_MGMT_2, 0x00);   // 加速度和陀螺仪都工作
        MPU_Set_Rate(100);                     // 最终采样率：100Hz
    }
}
```

**初始化流程图**：
```
复位 → 唤醒 → 设置量程(陀螺±2000/加±2g) → 200Hz自检 → 关中断/FIFO → 验证ID → 切PLL时钟 → 100Hz工作
```

### 辅助设置函数

```c
// 陀螺仪量程：0=±250, 1=±500, 2=±1000, 3=±2000 dps
u8 MPU_Set_Gyro_Fsr(u8 fsr);

// 加速度量程：0=±2g, 1=±4g, 2=±8g, 3=±16g
u8 MPU_Set_Accel_Fsr(u8 fsr);

// 采样率 = 1000 / (rate_div + 1)，自动设置 DLPF 为采样率的一半
u8 MPU_Set_Rate(u16 rate);   // rate=100 → 10Hz DLPF

// 低通滤波器截止频率
u8 MPU_Set_LPF(u16 lpf);     // 自动查表映射到寄存器值
```

### 原始数据读取（不使用 DMP）

```c
// 读 6 轴原始数据（ADC 值，非物理单位）
u8 MPU_Get_Gyroscope(short *gx, short *gy, short *gz);       // 陀螺仪
u8 MPU_Get_Accelerometer(short *ax, short *ay, short *az);   // 加速度计
// 本项目声明了这些函数但实际未使用，全部走 DMP
```

## 四、第3层：DMP 初始化 `MPU6050_DMP_Init()`

> DMP = Digital Motion Processor，MPU6050 内部的硬件协处理器，能自动融合 6 轴数据输出四元数，省去 MCU 做姿态解算的算力。

### 初始化序列（严格顺序，不能改）

```c
u8 MPU6050_DMP_Init(void) {
    MPU6050_IIC_IO_Init();                     // ① 重新初始化 I2C（确保总线干净）

    if (!mpu_init()) {                         // ② mpu_init()：底层硬件初始化
                                               //    - 复位传感器
                                               //    - 设置时钟源为 PLL
                                               //    - 设置采样率分频
                                               //    - 使能传感器

        mpu_set_sensors(INV_XYZ_GYRO | INV_XYZ_ACCEL);       // ③ 选择传感器：陀螺+加速度

        mpu_configure_fifo(INV_XYZ_GYRO | INV_XYZ_ACCEL);    // ④ 配置 FIFO 数据源

        mpu_set_sample_rate(DEFAULT_MPU_HZ);   // ⑤ 采样率 = 100Hz（#define DEFAULT_MPU_HZ 100）

        dmp_load_motion_driver_firmware();     // ⑥ ★ 加载 DMP 固件到 MPU6050 内部 RAM
                                               //    这是最关键的一步，将一大段二进制固件
                                               //    通过 I2C 写入 MPU6050 的 DMP 存储区

        dmp_set_orientation(                    // ⑦ 设置传感器安装方向矩阵
            inv_orientation_matrix_to_scalar(gyro_orientation));
            // gyro_orientation = {1,0,0, 0,1,0, 0,0,1} → 单位矩阵 = 默认安装方向

        dmp_enable_feature(                     // ⑧ 使能 DMP 功能
            DMP_FEATURE_6X_LP_QUAT       |     // 6轴低功耗四元数（核心输出）
            DMP_FEATURE_TAP              |     // 敲击检测
            DMP_FEATURE_ANDROID_ORIENT   |     // Android 方向检测
            DMP_FEATURE_SEND_RAW_ACCEL   |     // 原始加速度输出
            DMP_FEATURE_SEND_CAL_GYRO    |     // 校准后陀螺仪输出
            DMP_FEATURE_GYRO_CAL);              // 陀螺仪自动校准

        dmp_set_fifo_rate(DEFAULT_MPU_HZ);     // ⑨ FIFO 输出率 = 100Hz

        run_self_test();                       // ⑩ 自检：偏置校准写入 DMP

        mpu_set_dmp_state(1);                  // ⑪ ★ 使能 DMP（启动姿态解算）
    }
    return 0;
}
```

**DMP 初始化流程总结（11 步）**：
```
I2C初始化 → mpu_init → 选传感器 → 配FIFO → 设采样率100Hz
→ 加载DMP固件 → 设安装方向 → 使能DMP功能 → 设FIFO率 → 自检校准 → 使能DMP
```

### DMP 功能位掩码速查

| 宏 | 值 | 含义 |
|----|-----|------|
| `DMP_FEATURE_6X_LP_QUAT` | 0x010 | **6 轴低功耗四元数**（我们主要用这个） |
| `DMP_FEATURE_GYRO_CAL` | 0x020 | 陀螺仪自动校准 |
| `DMP_FEATURE_SEND_RAW_ACCEL` | 0x040 | 同时输出原始加速度 |
| `DMP_FEATURE_SEND_CAL_GYRO` | 0x100 | 同时输出校准后陀螺仪 |
| `DMP_FEATURE_TAP` | 0x001 | 敲击检测 |
| `DMP_FEATURE_ANDROID_ORIENT` | 0x002 | 屏幕方向检测 |

## 五、第4层：姿态数据获取 `MPU6050_DMP_Get_Data()`

### 完整数据流

```
MPU6050 内部 DMP
  │
  ├─ 硬件融合 6 轴数据（加速度+陀螺仪）
  ├─ 内部 FIFO 缓冲区（1024 字节）
  │
  └─ dmp_read_fifo()  ← 从 MPU6050 FIFO 读取原始四元数 (long quat[4])
       │
       ├─ 同时可获取 gyro[3], accel[3] 原始值
       │
       └─ 四元数归一化 → 欧拉角转换 ↓
```

### 核心代码逐行注解

```c
#define q30  1073741824.0f    // 2^30，DMP 四元数的定点数缩放因子

u8 MPU6050_DMP_Get_Data(float *pitch, float *roll, float *yaw)
{
    float q0=1.0f, q1=0.0f, q2=0.0f, q3=0.0f;       // 四元数初始化
    unsigned long sensor_timestamp;                    // 时间戳（本项目未用）
    short gyro[3], accel[3], sensors;                 // 原始传感器数据
    unsigned char more;                                // FIFO 是否还有剩余数据
    long quat[4];                                      // DMP 输出的原始四元数（q30格式）

    // ★ 步骤1：从 MPU6050 内部 FIFO 读取一帧数据
    if (dmp_read_fifo(gyro, accel, quat, &sensor_timestamp, &sensors, &more))
        return 1;    // 读取失败

    // ★ 步骤2：检查是否有四元数数据
    // INV_WXYZ_QUAT = 0x100，表示 FIFO 中包含四元数包
    if (sensors & INV_WXYZ_QUAT)
    {
        // ★ 步骤3：四元数从 q30 定点格式转为浮点数
        q0 = quat[0] / q30;    // quat[0] = w（实部）
        q1 = quat[1] / q30;    // quat[1] = x
        q2 = quat[2] / q30;    // quat[2] = y
        q3 = quat[3] / q30;    // quat[3] = z

        // ★ 步骤4：四元数 → 欧拉角（单位：度）
        // 公式来自四元数旋转矩阵的逆解
        *pitch = asin(-2 * q1 * q3 + 2 * q0 * q2) * 57.3;          // 俯仰角 ±90°
        *roll  = atan2(2 * q2 * q3 + 2 * q0 * q1,
                       -2 * q1 * q1 - 2 * q2 * q2 + 1) * 57.3;     // 横滚角 ±180°
        *yaw   = atan2(2 * (q1 * q2 + q0 * q3),
                       q0 * q0 + q1 * q1 - q2 * q2 - q3 * q3) * 57.3; // 偏航角 ±180°

        return 0;    // 成功
    }
    else
        return 2;    // FIFO 中没有四元数数据
}
```

**57.3 = 180/π**，弧度转角度的转换因子。

### 欧拉角输出范围

| 角度 | 范围 | 含义 |
|------|------|------|
| Pitch | -90° ~ +90° | 俯仰（点头），抬头为正 |
| Roll | -180° ~ +180° | 横滚（歪头），右倾为正 |
| Yaw | -180° ~ +180° | **偏航（转向）**，逆时针为正 |

> 本项目只使用 **Yaw** 做角度 PD 控制，Pitch 和 Roll 仅读取不控制。

## 六、主循环中如何使用 MPU6050

### 标准读取模式（main.c 中的核心模式）

```c
// 每个控制循环必须做的：
float Pitch, Roll, Yaw;
u8 ack;

// ★ 关键：循环重试直到成功
while (MPU6050_DMP_Get_Data(&Pitch, &Roll, &Yaw) != 0) {
    ack = 999;   // 失败标记（调试用，实际没用）
}
ack = 1;         // 成功标记

// 然后 Yaw 就可以用了
```

### Yaw 角 0~360° 转换（每个 step 里都有）

DMP 输出的 Yaw 范围是 **-180° ~ +180°**，本项目统一转为 **0° ~ 360°**：

```c
float Yaw0_360;
if (Yaw < 0)
    Yaw0_360 = Yaw + 360;     // -90° → 270°
else
    Yaw0_360 = Yaw;           // 90° → 90°

int yaw_int = (int)(Yaw0_360 + 0.5f);   // 四舍五入 → 整数角度
```

### 角度 PD 控制调用

```c
int pid_out = pid_angle2(目标角度, yaw_int);  // 如 pid_angle2(0,yaw_int) 保持0°
int aleft  = 30 - pid_out;
int aright = 30 + pid_out;
Limit(&aleft, &aright);
Set_Speed(aleft, aright);
```

> `pid_angle2` 原理：见 `循线控制逻辑详解.md` 或 `Cnotorl.c` 中的注释。
> 核心公式：`err = target - yaw`，取最短路径 (>180° 则 -360°，<-180° 则 +360°)，`out = ka2p*err + ka2d*diff`。

## 七、关键细节与注意事项

### 1. DMP 初始化可能失败（时序敏感）

```c
// main() 中的初始化：
MPU6050_Init();       // 基础初始化（几乎不会失败）
MPU6050_DMP_Init();   // DMP 初始化（可能失败！）
// ★ 注意：本项目没有检查 DMP 初始化返回值！
// 如果 dmp_load_motion_driver_firmware() 写入校验失败，
// 后续 DMP 读出的数据将是垃圾值。
```

**新项目建议**：加返回值检查 + 重试机制，或用 LED 闪烁提示失败状态。

### 2. DMP 读取必须循环重试

```c
// 不能这样写（可能读到旧数据或失败）：
MPU6050_DMP_Get_Data(&Pitch, &Roll, &Yaw);

// 正确写法：阻塞式重试
while (MPU6050_DMP_Get_Data(&Pitch, &Roll, &Yaw) != 0) {
    // 重试直到成功
}
```

原因：I2C 通信偶尔丢包、FIFO 数据未就绪、DMP 内部状态未更新等。**不重试会导致读到上一次的旧数据，造成角度控制抖动。**

### 3. OLED 和 MPU6050 共用 I2C 但引脚不同

| 设备 | SCL | SDA | 说明 |
|------|-----|-----|------|
| MPU6050 | PB6 | PB7 | 软件 I2C |
| OLED | PB10 | PB11 | 另一个软件 I2C |

它们使用**不同的 GPIO 引脚**，所以互不干扰。不是真正的总线共用，而是"两套独立的软件 I2C"。

### 4. 四元数的 q30 定点格式

DMP 输出的四元数使用 **Q30 定点数**：实际值 = 寄存器值 ÷ 2^30。

```c
#define q30  1073741824.0f    // = 2^30
q0 = quat[0] / q30;          // 比如 quat[0]=1073741824 → q0=1.0
```

### 5. 采样率设置

- `DEFAULT_MPU_HZ = 100` → 100Hz = 每 10ms 一帧数据
- FIFO 输出率 = 100Hz（`dmp_set_fifo_rate(100)`）
- 本项目主循环 `Delay_ms(10)` 也是 10ms，理论上刚好 1:1 匹配
- 实测由于 DMP 内部缓冲，`dmp_read_fifo` 调用频率可以略快于 100Hz

### 6. gyro_orientation 方向矩阵

```c
static signed char gyro_orientation[9] = {
    1, 0, 0,    // X轴：传感器X → 机体X（不变）
    0, 1, 0,    // Y轴：传感器Y → 机体Y（不变）
    0, 0, 1     // Z轴：传感器Z → 机体Z（不变）
};
```

这是**单位矩阵**，意味着 MPU6050 芯片方向与小车机体坐标系一致。如果你的安装方向不同（比如芯片转了个角度），需要修改这个矩阵。

### 7. 自检 run_self_test() 的作用

`run_self_test()` 调用 `mpu_run_self_test(gyro, accel)` 获取偏置值，然后写入 DMP：
```c
dmp_set_gyro_bias(gyro);     // 陀螺仪零偏写入 DMP
dmp_set_accel_bias(accel);   // 加速度零偏写入 DMP
```
这能让 DMP 输出的 Yaw 漂移更小。

## 八、调用链路全景图（在 main.c 中的位置）

```c
main()
  │
  ├─ MPU6050_Init()         ← 上电后调一次：唤醒芯片、设量程、验证ID
  ├─ MPU6050_DMP_Init()     ← 上电后调一次：加载DMP固件、配置、自检、使能
  │
  └─ while(1)
       │
       ├─ while(MPU6050_DMP_Get_Data(&P, &R, &Y) != 0) {ack=999;}
       │     │                                              ← 每10ms调一次
       │     └─ 读到 Yaw（-180°~+180°）
       │
       ├─ OLED_ShowSignedNum(1,1, Yaw, 5)  ← 显示 Yaw 在 OLED 第1行
       │
       └─ 根据 all_state 和 step 决定控制模式：
            │
            ├─ 【角度模式】（红外全白，走空地）
            │     Yaw → 0~360°转换
            │     pid_angle2(目标角度, yaw_int) → 差速直走
            │
            └─ 【循迹模式】（红外有线，跟着线走）
                  MPU6050 只用来显示 Yaw（不参与控制）
```

## 九、在新项目中使用的最小代码集

如果你想在新项目中只用 MPU6050 的 **Yaw 角度**（最常见的需求），以下是必须的文件清单：

| 必须 | 文件 | 原因 |
|------|------|------|
| ✅ | `MPU6050_I2C.c/h` | 软件 I2C 驱动 |
| ✅ | `mpu6050.c/h` | 寄存器操作 + 基础初始化 |
| ✅ | `inv_mpu.c/h` | DMP 初始化 + `MPU6050_DMP_Get_Data()` |
| ✅ | `inv_mpu_dmp_motion_driver.c/h` | DMP 固件数据 + `dmp_read_fifo()` |
| ✅ | `dmpKey.h` + `dmpmap.h` | DMP 固件的 key 和 memory map（编译依赖） |
| ❌ | `sys.c/h` | 只需 `sys.h` 中的位带操作宏 (`PBout`, `PBin`) 和 `My_GPIO_Init` |

### 需要根据你的硬件修改的地方

1. **I2C 引脚**（`MPU6050_I2C.h`）：
   ```c
   #define MPU6050_IIC_SCL_Pin    GPIO_Pin_X   // 改成你的 SCL 引脚
   #define MPU6050_IIC_SDA_Pin    GPIO_Pin_X   // 改成你的 SDA 引脚
   #define MPU6050_IIC_GPIO       GPIOX        // 改成你的 GPIO 端口
   ```

2. **I2C 延时**（`MPU6050_I2C.h`）：
   ```c
   #define MPU6050_IIC_delay_4us()   delay_us(4)   // 改用你的延时函数
   ```

3. **DMP 采样率**（`inv_mpu.h`）：
   ```c
   #define DEFAULT_MPU_HZ  (100)   // 100Hz，可按需调整
   ```

4. **`My_GPIO_Init` 函数**：要么移植 `sys.c` 中的实现，要么改为直接调用 `GPIO_Init()`。
